## 1. 问题背景

主服务器 `https://app.agentpit.io/explore` 和应用 `https://destiny.agentpit.io/` 位于相同主域名 `agentpit.io` 下的二级域名。用户在主站已登录的情况下，点击图片进入应用时，需要自动完成单点登录，无需用户再点击按钮授权。

之前的实现采用**弹窗式 OAuth2**，用户必须手动点击「agentpit 授权登陆」按钮才能触发授权流程。

## 2. 实现方案

在保留原有弹窗模式（手动登录）的基础上，新增**静默重定向 SSO 模式**：

- 弹窗模式（保留）：用户在登录页手动点击按钮，弹窗完成 OAuth，通过 `postMessage` 通知主窗口
- SSO 重定向模式（新增）：用户首次访问时自动全页面重定向到 AgentPit OAuth 授权页，如果用户已在主站登录且之前已授权过该应用，AgentPit 会自动回调（无需用户操作），实现静默登录

两种模式共享同一个 OAuth 回调地址 `/api/auth/agentpit/callback`，通过 `state` 参数区分。

## 3. 关键文件变更

### 后端

#### `service/AgentpitOAuthService.java`
- 新增 `buildAuthorizeUrl(String state)` 重载方法，支持传入 OAuth `state` 参数

#### `controller/AgentpitOAuthController.java`
- 新增 `GET /api/auth/agentpit/sso?returnUrl=/` SSO 入口端点
  - 将 `returnUrl` 编码到 `state` 参数中（格式：`sso:/path`）
  - 302 重定向到 AgentPit 授权页
- 修改 `GET /api/auth/agentpit/callback` 回调端点
  - 通过 `state` 参数前缀 `sso:` 判断是 SSO 模式还是弹窗模式
  - SSO 模式：返回 HTML 页面，通过 JS 跳转到前端 SSO 回调页（`/auth/sso/callback#token=xxx&user=xxx`）
  - 弹窗模式：保持原有 `postMessage` 行为不变
- Token 通过 URL hash（`#`）传递而非 query params，避免 token 出现在服务器日志中

### 前端

#### `pages/SsoCallbackPage.jsx`（新建）
- SSO 回调页，从 URL hash 中提取 `token` 和 `user` 信息
- 调用 `auth.login()` 完成登录
- 登录成功后跳转到原始页面（从 `returnUrl` 参数获取）
- 登录失败跳转到 `/login` 页

#### `utils/ssoHelper.js`（新建）
- `shouldAutoSso()`：判断是否应该自动触发 SSO
  - 排除 SSO 回调页和登录页
  - 排除 URL 带 `sso_error` 参数的情况
  - 排除本次 session 已尝试过 SSO 的情况
- `markSsoAttempted()`：标记本次 session 已尝试 SSO
- `clearSsoAttempted()`：登录成功后清除标记

#### `context/AuthContext.jsx`
- 在 `initializeAuth` 中，当没有本地 token 时，调用 `shouldAutoSso()` 判断是否自动触发 SSO
- 条件满足时：标记 `sso_attempted`，全页面重定向到 `/api/auth/agentpit/sso?returnUrl=当前路径`
- 登录成功时调用 `clearSsoAttempted()` 清除标记

#### `pages/LoginPage.jsx`
- 新增对 URL 参数 `sso_error` 的处理，显示 SSO 失败的错误信息

#### `App.jsx`
- 新增路由 `/auth/sso/callback` 指向 `SsoCallbackPage`

## 4. SSO 完整流程

```
用户从 app.agentpit.io 点击进入 destiny.agentpit.io
  |
  v
AuthContext 初始化，检查本地 token
  |
  ├── 有 token → 正常验证登录（原有逻辑）
  |
  └── 无 token → shouldAutoSso() 检查
        |
        ├── 不满足条件 → 正常显示页面（未登录状态）
        |
        └── 满足条件 → markSsoAttempted()
              |
              v
        重定向到 /api/auth/agentpit/sso?returnUrl=/当前路径
              |
              v
        后端 302 重定向到 AgentPit 授权页（state=sso:/当前路径）
              |
              ├── 用户已登录且已授权 → AgentPit 自动回调（无需用户操作）
              |
              ├── 用户已登录未授权 → 显示授权确认页
              |
              └── 用户未登录 → 显示登录页
              |
              v
        回调 /api/auth/agentpit/callback?code=xxx&state=sso:/路径
              |
              v
        后端处理 code，生成 JWT
              |
              v
        返回 HTML 页面，JS 跳转到 /auth/sso/callback#token=xxx&user=xxx
              |
              v
        SsoCallbackPage 解析 hash，调用 auth.login()
              |
              v
        登录成功，跳转到原始页面
```

## 5. 防无限循环机制

使用 `sessionStorage` 中的 `sso_attempted` 标记防止无限重定向：

| 场景 | 行为 |
|------|------|
| 首次访问，无 token | 触发一次 SSO 重定向 |
| SSO 成功 | 清除标记，正常使用 |
| SSO 失败（用户未在主站登录） | 跳转到 `/login?sso_error=xxx`，不再重试 |
| 用户刷新页面（同一 session） | 标记已存在，不触发 SSO |
| 用户新开浏览器标签 | 新 session，会再次尝试 SSO |
| 访问 `/login` 或 `/auth/sso/callback` | 不触发 SSO |

本次修改涉及前端和后端，需要同时重建两个容器。

```bash
  src/main/java/com/example/demo/service/AgentpitOAuthService.java \
  src/main/resources/application.yml \
  src-frontend/pages/SsoCallbackPage.jsx \
  src-frontend/pages/LoginPage.jsx \
  src-frontend/utils/ssoHelper.js \
  src-frontend/context/AuthContext.jsx \
  src-frontend/App.jsx \
  pom.xml

# 2. 上传到服务器
scp -i ~/.ssh/id_rsa_google_longterm /tmp/baziback-sso.tar.gz a1@35.198.208.41:/tmp/

# 3. 服务器操作
ssh -i ~/.ssh/id_rsa_google_longterm a1@35.198.208.41
cd /opt/baziback-docker/app
tar xzf /tmp/baziback-sso.tar.gz

# 4. 编译后端 JAR
docker run --rm \
  -v "$(pwd)":/workspace \
  -w /workspace \
  maven:3.9-eclipse-temurin-17 \

# 5. 重建前端和后端容器
docker compose up -d --force-recreate --build

docker compose ps
docker compose logs backend --tail=30
docker compose logs frontend --tail=30
```

## 7. 前提条件与注意事项

### AgentPit OAuth 静默授权支持

SSO 自动登录依赖 AgentPit OAuth 对**已授权应用的静默回调**：
- 用户已登录 AgentPit 且之前已授权过该应用时，OAuth 服务器直接回调，不再显示授权确认页
- 大多数 OAuth2 提供者默认支持此行为
- 如果 AgentPit 每次都要求用户手动确认，需要联系 AgentPit 确认是否支持 `prompt=none` 或类似参数

### 安全考虑

- Token 通过 URL hash（`#`）传递，不会发送到服务器，不会出现在 Nginx/Caddy 访问日志中
- SSO 回调页读取 hash 后立即通过 `window.history.replaceState` 清除 URL 中的敏感信息
- 每个 session 最多触发一次自动 SSO，避免被用于循环攻击

### 兼容性

- 原有的弹窗式 OAuth 登录完全保留，用户仍可在登录页手动点击按钮登录
- SSO 仅在无 token 且未尝试过时自动触发，不影响已登录用户
- `state` 参数通过前缀 `sso:` 区分模式，不影响原有无 state 的弹窗回调
